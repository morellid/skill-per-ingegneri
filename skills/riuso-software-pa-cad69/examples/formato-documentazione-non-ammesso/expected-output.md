# Output atteso - formati della documentazione (A.8)

## Documentazione tecnica: no a PDF/DOCX/ODT

- Per la **documentazione tecnica** allegata al software, le Linee guida AgID (Allegato A,
  **A.8**) richiedono un **formato testuale che garantisca il versionamento riga per
  riga**. Sono ammessi: **HTML, Markdown, reStructuredText, LaTeX**.
- La documentazione in **ODT, DOCX o PDF NON è ammessa**, perché con questi formati non è
  possibile definire le versioni «riga per riga».
- La documentazione deve inoltre seguire le indicazioni sul rilascio di documentazione
  tecnica delle **Linee Guida di design per i servizi web della PA** e della **Guida a
  Docs Italia** (AgID).

## Manuale utente: eccezione

- Se il capitolato prevede anche un **manuale utente** (documentazione per gli utenti
  finali), l'obbligo di rilascio si estende ad esso; per questa documentazione sono
  consentiti **anche formati binari, purché aperti, modificabili e multipiattaforma** —
  resta comunque **escluso il PDF**.

## Quindi

- Documentazione tecnica in **PDF/DOCX**: **non ammessa** → convertire in Markdown/HTML/
  reST/LaTeX.
- Manuale utente: formato aperto/modificabile ammesso, **ma non PDF**.
